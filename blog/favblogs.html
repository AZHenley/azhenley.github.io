Technical blogs worth reading

7/29/2021

I'm dedicated to checking my favorite blogs every few days (and always looking for more), so I was surprised when a group of friends said they don't know of a single blog worth reading!

This list is so that I can easily share my favorite blogs to colleagues and students. These blogs cover programming, software engineering, startups, product design, usability, research, game development, and academia. They are in no particular order, but I provide a few categories for each and an example post that I recommend.



  • Eli Bendersky's Blog (eli.thegreenplace.net)
    • parsing, Go, cpython, programming
    • Ex: Python internals: adding a new statement to Python
  • Where Did All the Time Go? by Bob Nystrom (stuffwithstuff.com)
    • compilers, game development, programming
    • Ex: Rooms and Mazes: A Procedural Dungeon Generator
  • Embedded in Academia by John Regehr (regehr.org)
    • compilers, C, academia
    • Ex: The Little C Function From Hell
  • Hillel Wayne's Blog (hillelwayne.com)
    • formal methods, programming languages, programming
    • Ex: Why do interviewers ask linked list questions?
  • Tiny Projects by Ben Stokes (tinyprojects.dev)
    • entrepreneurship, products
    • Ex: Selling Tiny Internet Projects For Fun and Profit
  • Coding Horror by Jeff Atwood (codinghorror.com)
    • software engineering, products, technology
    • Ex: The PHP Singularity
  • Joel on Software by Joel Spolsky(joelonsoftware.com)
    • software engineering, products, startups
    • Ex: The Law of Leaky Abstractions
  • Dan Luu's Blog (danluu.com)
    • systems, programming
    • Ex: What's new in CPUs since the 80s?
  • Paranoid Coding by Jared Par (paranoidcoding.com)
    • compilers, C#, programming
    • Ex: The File System is Unpredictable
  • Phil Eaton's Blog (eatonphil.com)
    • compilers, databases, programming
    • Ex: Writing a SQL database from scratch in Go: 1. SELECT, INSERT, CREATE and a REPL
  • Ben Hoyt's Blog (benhoyt.com)

  • Daniel Lemire's Blog (lemire.me)
